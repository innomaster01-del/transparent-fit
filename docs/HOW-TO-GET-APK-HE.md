# איך להשיג APK מותקן בטלפון — בלי מחשוב מיוחד, בלי Android Studio

הפרויקט כולל בנייה אוטומטית בענן (GitHub Actions). כל מה שצריך זה חשבון GitHub חינמי.

## שלב 1 — פתח חשבון GitHub (אם אין לך)
1. גלוש ל-https://github.com/signup
2. הרשמה עם אימייל — חינם

## שלב 2 — צור repository חדש
1. לחץ על הפלוס (+) למעלה מימין → **New repository**
2. שם: `transparent-fit` (או כל שם)
3. בחר **Private** (שאף אחד לא יראה את הקוד)
4. לחץ **Create repository**

## שלב 3 — העלה את הפרויקט
הדרך הקלה בלי שורת פקודה:
1. חלץ את ה-ZIP של הפרויקט במחשב
2. בעמוד ה-repository החדש לחץ **uploading an existing file**
3. גרור את **כל** התיקיות והקבצים מתוך תיקיית TransparentFit
   (חשוב: את התוכן של TransparentFit, לא את התיקייה עצמה)
4. למטה לחץ **Commit changes**

⚠️ שים לב: התיקייה `.github` היא תיקייה מוסתרת — ודא שהיא עלתה!
אם הגרירה לא כללה אותה, השתמש ב-GitHub Desktop או העלה אותה בנפרד.

## שלב 4 — הפעל את הבנייה
1. בעמוד ה-repository לחץ על הלשונית **Actions**
2. אם מופיעה הודעה על הפעלת workflows — אשר (Enable)
3. בצד שמאל בחר **Build Android APK**
4. לחץ על הכפתור **Run workflow** → **Run workflow** (ירוק)
5. המתן 10–15 דקות (יופיע עיגול צהוב מסתובב, ואז ✅ ירוק)

## שלב 5 — הורד את ה-APK
1. לחץ על הריצה שהסתיימה (השורה עם ה-✅)
2. גלול למטה לאזור **Artifacts**
3. לחץ על **TransparentFit-APK** — יורד קובץ ZIP
4. חלץ ממנו את `app-release.apk`

## שלב 6 — התקן בטלפון
1. העבר את ה-APK לטלפון (WhatsApp לעצמך / כבל / Google Drive)
2. פתח את הקובץ בטלפון
3. אנדרואיד יבקש לאשר "התקנה ממקורות לא ידועים" — אשר
4. לחץ התקן — וזהו! האפליקציה על המסך

## בעיות נפוצות
| בעיה | פתרון |
|------|-------|
| אין לשונית Actions | Settings → Actions → General → Allow all actions |
| הבנייה נכשלה (X אדום) | לחץ על הריצה → לחץ על השלב האדום → צלם את השגיאה ושלח לי |
| הטלפון חוסם התקנה | הגדרות → אבטחה → אפשר התקנת אפליקציות לא ידועות |
| אין תיקיית .github ב-repo | היא תיקייה נסתרת — העלה ידנית: Create new file → הדבק את הנתיב `.github/workflows/build-android-apk.yml` ואת התוכן |

## מה ה-APK הזה ומה הוא לא
- ✅ אפליקציה מלאה מותקנת — כל המסכים, מצלמה, כל 4 המצבים
- ✅ חתום בחתימת debug — מתאים להתקנה ידנית ולבדיקות
- ❌ לא מתאים להעלאה ל-Google Play כמו שהוא — לזה צריך חתימה רשמית
