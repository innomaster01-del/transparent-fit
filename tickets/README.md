# Transparent Fit — Build Tickets

The work is split into 6 phases, ~33 working days for one experienced React Native
developer. Each ticket below is a self-contained chunk of work with clear acceptance
criteria. Phases must run in order; tickets within a phase can sometimes parallelize
if two devs are working.

## Phase summary

| # | Phase | Tickets | Days |
|---|---|---|---|
| 1 | Foundation: native modules + camera | T1.1 – T1.3 | 5 |
| 2 | Onboarding flow (Screens 1–3) | T2.1 – T2.3 | 6 |
| 3 | Live try-on (Screen 4) | T3.1 – T3.3 | 8 |
| 4 | Polish (Screen 5, Settings, perf) | T4.1 – T4.3 | 4 |
| 5 | QA + beta | T5.1 – T5.2 | 5 |
| 6 | Store submission | T6.1 – T6.2 | 5 |
